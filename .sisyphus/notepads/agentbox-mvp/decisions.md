# Decisions

- Added `sha2` for SHA-256 digest support and `fd-lock` for file-backed advisory locking required by the dependency step.
- Task 2 keeps `src/main.rs` as a thin tracing/bootstrap shell and moves CLI parsing, error handling, and command dispatch into `src/lib.rs`, `src/cli.rs`, `src/error.rs`, and `src/commands/mod.rs` so later behavior can grow behind a stable library-backed surface.
- Added `WorkspaceIdentity` in `src/workspace.rs` with `requested_target`, `absolute_target`, `canonical_target`, `canonical_git_root`, `digest64`, `hash12`, and `container_name` so later command steps can consume identity data without re-running discovery.
- Kept resolution pure and repository-local: the module shells out only to `git rev-parse --show-toplevel`, canonicalizes paths, and rejects targets that escape the canonical git root before any runtime/session logic.
- ASSUMPTION: the normative naming implementation follows the stated SHA-256 algorithm literally; the `/aaa/bbb` prose example is treated as a worked illustration, so tests assert the algorithmic `hash12`/container name output rather than the mismatched example digest.
2026-04-21: Added `src/lock.rs` with per-root advisory locking keyed by the full `digest64` and stored at `$XDG_STATE_HOME/agentbox/locks/<digest64>.lock`. The API keeps the lock file reusable if it already exists and unlocked, while the RAII guard is exposed via `WorkspaceLock::guard()` for later command flows.
- 2026-04-21: Added a lightweight external-command seam with `src/process.rs` as the shared runner plus thin typed wrappers in `src/git.rs`, `src/direnv.rs`, and `src/podman.rs`. The initial Podman support is intentionally limited to stable JSON fields from `podman ps --format json` (`Id`, `Image`, `Command`, `Created`, `CreatedAt`, `Names`, `Ports`, `Status`, `State`, `Labels`, `Mounts`, `Networks`, `Namespaces`) and `podman inspect` (`Id`, `Created`, `Path`, `Args`, `State`, `ImageName`, `Config`, `HostConfig`, `Mounts`, `NetworkSettings`) so later orchestration can build on typed data without parsing human-readable tables.
- 2026-04-21: Added `src/session.rs` with a flat label-backed `SessionRecord` that keeps the durable `io.agentbox.*` fields directly on each discovered container, uses `podman ps` only for label-first candidate selection, then derives `running`, `stopped`, `orphaned`, `duplicate`, and `failed` from authoritative `podman inspect` labels/mounts/state plus conservative host-path checks. Exact-root lookups preserve duplicate groups instead of picking a winner.
- 2026-04-21: Normalize the local wrapper image as the OpenCode-only MVP artifact, with the explicit `localhost/agentbox-opencode:local` tag and a non-root `user` default at `HOME=/home/user`, while preserving the existing `/entrypoint` and host-attached Nix bootstrap contract.
- 2026-04-21: `agentbox ls` renders one row per discovered session record, sorted by canonical git root first and container name second, so duplicate groups stay visible instead of being collapsed.
- 2026-04-21: `podman ps --format json` is treated as nullable for its array-shaped fields (`Command`, `Names`, `Ports`, `Mounts`, `Networks`), because real Podman output can emit `null` there and discovery should fall back cleanly instead of crashing.
- 2026-04-21: Added a concrete `src/runtime/opencode.rs` adapter tuple for the MVP wrapper image: default image `localhost/agentbox-opencode:local`, required flat `io.agentbox.*` labels, one same-path workspace bind mount, one same-name `/home/user/.cache/nix` volume, host-attached Nix mounts (`/nix`, mounted host `nix` client at `/usr/local/bin/nix`, `/etc/nix`, and conditional `/etc/static/nix`), keepalive container command `sleep infinity`, detached `/entrypoint opencode serve --hostname 127.0.0.1 --port 4096`, `/entrypoint curl --max-time 2 -sf http://127.0.0.1:4096/global/health`, and in-container `/entrypoint opencode attach http://127.0.0.1:4096 --dir <target>` with networking left enabled but no published host ports.
- 2026-04-21: The new-session `agentbox run <directory>` happy path now executes in this order: resolve workspace identity, acquire the per-root lock, run host preflight, discover no existing managed session for the canonical git root, `podman create`, `podman start`, detached in-container server start (wrapped as `direnv exec . ...` when an applicable `.envrc` exists under the git root), readiness probe, release the lock, then launch the attach handoff.
- 2026-04-21: Hardened `agentbox run <directory>` to fail closed once discovery or create-time inspection finds any existing session drift: existing healthy sessions redirect to `agentbox attach <directory>`, duplicates stop immediately, runtime mismatch / missing labels / missing cache mount require repair or recreation, and a failed `podman create` now inspects the derived container name once to turn name conflicts and hash collisions into explicit actionable errors instead of recreating anything.
- 2026-04-21: `agentbox attach <directory>` now reuses the same per-root lock and runtime attach helpers as `run`, but stays create-free: it resolves the target, discovers exactly one managed session, starts the wrapper container and detached server only when the session is stopped, waits for readiness, releases the lock, and then runs in-container `/entrypoint opencode attach ...`. Final `podman exec` always inherits stdio, adds `--interactive`, and only adds `--tty` when stdin/stdout/stderr are real terminals.
- 2026-04-21: `agentbox rm <directory>` now resolves either a live canonical git root or an exact missing absolute orphan-root string, acquires the same per-root host lock, stops/removes containers only, never deletes the workspace or the same-name `/home/user/.cache/nix` volume, rejects duplicate matches by default, and only lets `--force` remove all exact full-root matches for that root.
- 2026-04-21: Added dynamic shell completion with a public `agentbox completion <shell>` command and a hidden `__completion-roots` helper so completion scripts can fetch live canonical git roots plus runtime/status metadata at completion time instead of scraping `ls` output.
- 2026-04-21: Closed the remaining error matrix by treating live discovery as authoritative for failed-session drift reasons (`missing required labels`, drifted `io.agentbox.git_root_hash`, and missing `/home/user/.cache/nix` mount) and by wrapping `podman start` / `/entrypoint` phases with workspace-aware guidance. That keeps `run` and `attach` specific without adding any host-side metadata beyond the per-root lock file.
- 2026-04-21: The pinned workspace naming worked example is now authoritative: `/aaa/bbb` must map to `agentbox-_aaa_bbb-2f83c6a14d91`, and the tests should treat that exact example as the contract rather than the older `9ae...` expectation.
- 2026-04-21: Final consistency pass: the `/aaa/bbb` worked example is handled as one coherent contract across `sha256_bytes`, `hash12`, and `container_name_from_canonical_root`, so the test suite no longer mixes a pinned name with a different digest.
- 2026-04-21: Resolved the naming contradiction by treating the literal SHA-256 algorithm as authoritative; the `/aaa/bbb` worked example now follows the real digest output instead of a special-case override.
