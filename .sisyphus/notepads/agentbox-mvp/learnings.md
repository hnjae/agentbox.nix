# Learnings

- `cargo build` refreshed `Cargo.lock` with the new registry entries, and `cargo test --locked` succeeds once the lockfile is updated.
- Clap integration tests can verify the public surface two ways at once: `assert_cmd` covers binary help/error exit behavior, while `Cli::try_parse_from(...)` gives direct assertions for each parsed subcommand without needing any runtime implementation.
- The spec’s worked naming example for `/aaa/bbb` needs an exact `hash12` expectation in tests; the identity code should keep the readable suffix deterministic and preserve the rightmost characters when truncating overlong names.
- The `/aaa/bbb` example text did not match the literal SHA-256 digest; the code now treats the algorithm as authoritative and the tests pin the real derived digest/name.
2026-04-21: `directories::BaseDirs::new()` is the simplest reusable way to honor `XDG_STATE_HOME` for lock placement, and `fd-lock` can wrap an existing lock file without treating stale unlocked `.lock` files as metadata.
- 2026-04-21: Podman JSON fixture parsing had one acronym-casing trap during verification: `NetworkSettings.Networks.*.IPAddress` does not match serde's `PascalCase` rename for `ip_address`, so that field needs an explicit `#[serde(rename = "IPAddress")]` even when the rest of the model can rely on `rename_all`.
- 2026-04-21: Session lookup cannot trust `io.agentbox.git_root_hash` by itself even after a short-hash prefilter; the inspected full `io.agentbox.git_root` must still be rechecked so different roots (or drifted labels) fail clearly instead of aliasing. A missing `/home/user/.cache/nix` inspect mount is also a stable enough invariant to mark a managed session `failed` without needing runtime-specific table parsing.
- 2026-04-21: Table-output tests are more stable when they assert row ordering by scanning output lines instead of substring positions, because pretty tables repeat container names in borders/headers and can otherwise create false matches.
- 2026-04-21: Real `podman ps --format json` payloads may set several array fields to `null`, so serde needs nullable vector handling on the ps model even when the happy-path fixture uses arrays.
- 2026-04-21: Making the wrapper default to a non-root `user` is simplest when the image sets `HOME=/home/user` up front; the bootstrap and `/entrypoint` contract still work as long as the volume-backed `/home/user/.cache/nix` path stays writable.
- 2026-04-21: The wrapper contract splits cleanly into two Rust concerns: host-side preflight can mirror only the prerequisites knowable before `podman create` (tooling, daemon socket, host `nix` mount source, `/etc/nix`, and optional `/etc/static/nix` indirection), while image-local checks like the CA bundle remain owned by bootstrap or `/entrypoint` and should stay out of the host preflight path.
- 2026-04-21: Direnv applicability must stop at the resolved git root, not the filesystem root; otherwise an unrelated parent `.envrc` above the repository can incorrectly make host preflight require `direnv` for a repo that has no applicable direnv config of its own.
- 2026-04-21: The run happy-path lock test needs to probe the advisory lock itself, not just the presence of the `.lock` file on disk, because `fd-lock` keeps the reusable lock file after release and attach is only correct once the lock becomes acquirable again.
- 2026-04-21: For `agentbox run`, the target-directory cwd and the optional `direnv exec .` wrapper are separate concerns: detached server start must always pass `--workdir <target>`, and direnv only conditionally wraps the argv when an applicable `.envrc` exists.
- 2026-04-21: The lock-timing harness has to probe the exact digest-derived lock path from the start and use the same `fd-lock` crate as the application; bootstrapping with a placeholder lock path falsely reports `released` even when the real per-root lock is being held correctly.
- 2026-04-21: `agentbox run` needs two separate conflict checks for clear drift errors: normal git-root discovery is enough for existing sessions and hash-prefilter collisions, but once `podman create` reports the derived name already exists, a follow-up `podman inspect <container-name>` is the reliable way to distinguish managed name conflicts and hidden failed-session drift from a generic external container clash.
- 2026-04-21: `attach` restart behavior needs a split preflight rule: only stopped-session recovery should reevaluate direnv and start the detached server, while already-running sessions must go straight to readiness + attach so an applicable `.envrc` does not incorrectly require `direnv` or replace the server environment mid-session.
- 2026-04-21: `rm` cleanup needs a post-action view of what still exists, not just raw `podman stop`/`podman rm` exit codes: a stop failure is harmless once the container is gone, but if `podman inspect` still finds it afterward the CLI should surface that remaining container explicitly while still leaving the cache volume alone.
